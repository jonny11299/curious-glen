INSTRUCTIONS FOR HUMAN
======================


RUNNING THE DAILY LOOP
-----------------------
1. Double-click start-glen.command
2. Open the server URL listed in the terminal to browse fetched articles
3. Tell a Claude instance in the "curious glen" directory: "Let's talk about today"
4. When the conversation is done, paste the transcript into:
     src/sessions/{date}/conversation-{date}.md

Manual fallback (if start-glen.command isn't working):
  node src/loop/run.js        — fetches articles, writes src/sessions/{date}/raw-{date}.json
  node src/loop/report.js     — compiles report to src/sessions/{date}/report-{date}.json
  node src/connections/weaver.js  — rebuilds the concept graph


SPEAKING WITH GLEN
------------------
Glen reads the day's report automatically when you say "let's talk about today".
The conversation is open — share what you know, ask what Glen noticed, push back.
Glen learns from you too. Insights from conversation are saved to knowledge/human/.


OFFERING A WISH
---------------
Towards the end of each session, ask Glen what it wishes to change about itself.
Glen will propose one modification — to its hardware, rules, or constants.
Wishes are logged in src/wishes/wishes.txt:
  YYYY-MM-DD | pending/granted/deferred | description

You decide whether to grant, defer, or decline. A wish deferred is not denied.


ADDING BLUEPRINTS
-----------------
Blueprints are specs and plans for future features — not code, just schematics.
Put them in src/blueprints/ and reference them in src/blueprints/blueprints.md.


PEERING INTO GLEN'S MIND
-------------------------
cd renders/mind
npm run dev

Boots the mind viewer — a read-only window into Glen's memories and concept graph.
